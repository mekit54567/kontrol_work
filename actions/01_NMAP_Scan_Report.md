# Отчёт 01 — Сканирование Nmap и анализ сервисов

## Цель
Сканирование учебной виртуальной машины (Metasploitable) для выявления открытых портов и сетевых сервисов.

## Окружение
- Машина-цель: Metasploitable (пример IP в лаборатории: `10.0.0.10`)
- Сканирующая машина: Kali / любая Linux-машина с nmap

## Команды (пример)
```bash
# Быстрое определение хостов и открытых портов
nmap -sS -Pn -p- 10.0.0.10 -oN results/nmap_full_tcp.txt

# Версия сервисов и скрипты NSE
nmap -sV -sC -p22,21,23,80,139,445,3306 10.0.0.10 -oN results/nmap_services.txt

# UDP скан (медленный)
sudo nmap -sU -p- 10.0.0.10 -oN results/nmap_udp.txt
```

## Примерный вывод (результат сохранён в `results/nmap_output.txt`)
- Открытые TCP-порты: 21 (ftp), 22 (ssh), 23 (telnet), 80 (http), 139/445 (smb), 3306 (mysql) и др.
- Версии сервисов: vsftpd 2.3.4, OpenSSH 4.x, Apache 2.2.x, MySQL 5.x

## Поиск известных уязвимостей
- Для каждой найденной службы смотрим Exploit-DB по названию и версии (например, `vsftpd 2.3.4 exploit`).
- Ссылки на найденные эксплоиты (пример):
  - vsftpd 2.3.4 — https://www.exploit-db.com/exploits/1744
  - vsftpd v2.3.4 backdoor — https://www.exploit-db.com/exploits/1746

> **Команда для автоматического поиска CVE / эксплоитов** (локально, вручную):
```bash
# Поиск по имени сервиса в Exploit-DB (пример для vsftpd)
# Открываем веб-интерфейс exploit-db.com и вводим в поиске "vsftpd 2.3.4"
```

## Выводы и рекомендации
- Обновлять сервисы, закрывать ненужные порты, применять брандмауэры.
- Для отчётности: сохранить `results/nmap_output.txt` и добавить ссылки на эксплоиты.
